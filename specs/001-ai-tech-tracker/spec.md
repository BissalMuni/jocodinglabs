## Clarifications

### Session 2026-02-28

- Q: 실습 데이터 저장 위치? → A: 브라우저 로컬 저장 (localStorage). 기기별 독립, 인증 불필요
- Q: LLM 자동 추출 범위? → A: 이름 + 설명 + 카테고리 자동 분류 (난이도는 수동)
- Q: 영상 분석 방식? → A: 단건 기본 + 일괄 처리 (여러 URL 순차 분석)
- Q: 키워드 검색 기능? → A: 기술명/설명 대상 텍스트 검색 포함
- Q: 초기 카테고리 목록? → A: 이미지 생성, 영상 생성, 음성/음악 AI, 코딩 도구, 챗봇/에이전트, 생산성 도구, 기타 (7개)

### Session 2026-03-02

- Q: 카테고리 초기 데이터? → A: 고정 7개 대신 AI 분석 시 동적 생성. 기존 카테고리 있으면 재사용, 없으면 신규 추가
- Q: 기술 항목에 URL 포함? → A: 각 기술의 공식 홈페이지 URL 포함. 자막에서 추출하지 않고 AI가 별도 조사
- Q: 채널 영상 수집 방식? → A: 조코딩 채널 /search?query=AI뉴스 페이지 스크래핑으로 자동 수집
- Q: 기술 목록 표시 방식? → A: 영상별 그룹핑 (최신순) + 기술 카테고리 필터 이중 분류
- Q: 주간 자동화? → A: Vercel Cron으로 매주 자동 sync+analyze, 기처리 영상 제외

## User Scenarios & Testing *(mandatory)*

### User Story 1 - AI 기술 목록 조회 (Priority: P1)

사용자는 웹사이트에 접속하여 조코딩 AI 뉴스 영상에서 추출된
AI 기술 목록을 한눈에 확인할 수 있다. 각 항목에는 기술 이름,
간단한 설명, 출처 영상 링크, 소개된 날짜가 표시된다.
사용자는 카테고리 필터, 실습 상태 필터, 텍스트 검색을 통해
원하는 기술을 빠르게 찾을 수 있다.

**Why this priority**: 프로젝트의 핵심 가치는 기술 목록을
쉽게 조회하는 것이다. 이 기능 없이는 다른 기능이 의미 없다.

**Independent Test**: 웹사이트에 접속하여 기술 목록이 표시되는지
확인하고, 각 항목의 출처 영상 링크가 정상 동작하는지 클릭하여 검증.

**Acceptance Scenarios**:

1. **Given** 웹사이트에 등록된 AI 기술이 10개 이상 있을 때, **When** 사용자가 메인 페이지에 접속하면, **Then** 최신순으로 정렬된 기술 목록이 표시된다
2. **Given** 기술 목록이 표시된 상태에서, **When** 사용자가 특정 기술의 출처 영상 링크를 클릭하면, **Then** 해당 YouTube 영상 페이지로 이동한다
3. **Given** 기술 목록이 표시된 상태에서, **When** 사용자가 카테고리 필터를 선택하면, **Then** 해당 카테고리의 기술만 표시된다
4. **Given** 기술 목록이 표시된 상태에서, **When** 사용자가 검색창에 키워드를 입력하면, **Then** 기술명 또는 설명에 해당 키워드가 포함된 항목만 표시된다

---

### User Story 2 - 자막 기반 기술 자동 추출 (Priority: P2)

관리자는 조코딩 채널의 AI 뉴스 영상 URL을 입력하면,
시스템이 자동으로 자막을 추출하고 LLM을 통해 영상에서
소개된 AI 기술 목록(이름, 설명, 카테고리)을 자동 생성한다.
단건 URL 입력과 여러 URL 일괄 입력을 모두 지원한다.

**Why this priority**: 수동 입력 대비 효율을 크게 높이는 핵심
자동화 기능이다. Constitution의 Transcript-First Analysis 원칙 구현.

**Independent Test**: AI 뉴스 영상 URL 하나를 입력하고,
자막 추출 및 LLM 분석을 거쳐 기술 목록이 자동 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 관리자가 자막이 있는 AI 뉴스 영상 URL을 입력했을 때, **When** 분석 요청을 실행하면, **Then** 영상에서 소개된 AI 기술의 이름, 설명, 카테고리가 목록으로 추출된다
2. **Given** 자동 추출된 기술 목록이 표시된 상태에서, **When** 관리자가 각 항목을 확인/수정 후 저장하면, **Then** 해당 기술들이 메인 목록에 추가된다
3. **Given** 자막이 없는 영상 URL을 입력했을 때, **When** 분석 요청을 실행하면, **Then** "자막을 찾을 수 없습니다" 메시지가 표시된다
4. **Given** 관리자가 여러 영상 URL을 일괄 입력했을 때, **When** 분석 요청을 실행하면, **Then** 각 영상이 순차적으로 분석되고 개별 결과가 표시된다

---

### User Story 3 - 실습 상태 추적 (Priority: P3)

사용자는 목록의 각 AI 기술에 대해 실습 상태(미시작/진행중/완료)를
표시하고, 실습 메모를 기록할 수 있다. 실습 데이터는 브라우저
로컬 저장소에 저장되어 인증 없이 사용 가능하다.

**Why this priority**: Hands-On First 원칙의 핵심 기능이지만,
기술 목록 조회와 자동 추출이 먼저 있어야 추적할 대상이 존재한다.

**Independent Test**: 특정 기술 항목의 실습 상태를 "진행중"으로
변경하고 메모를 작성한 뒤, 페이지를 새로고침하여 유지되는지 확인.

**Acceptance Scenarios**:

1. **Given** 기술 목록에서 특정 항목을 선택했을 때, **When** 사용자가 실습 상태를 "진행중"으로 변경하면, **Then** 해당 상태가 브라우저 로컬에 저장되고 목록에 반영된다
2. **Given** 실습 상태가 설정된 항목에서, **When** 사용자가 실습 메모를 작성하고 저장하면, **Then** 메모가 브라우저 로컬에 저장된다
3. **Given** 기술 목록이 표시된 상태에서, **When** 사용자가 실습 상태별 필터를 적용하면, **Then** 선택한 상태의 항목만 표시된다
4. **Given** 사용자가 페이지를 새로고침하거나 재접속했을 때, **When** 동일한 브라우저에서 접속하면, **Then** 이전에 저장한 실습 상태와 메모가 그대로 유지된다

---

### User Story 4 - 기술 목록 관리 (Priority: P4)

관리자는 기술 항목을 수동으로 추가, 수정, 삭제할 수 있다.
자동 추출 외에도 직접 항목을 관리할 수 있어야 한다.

**Why this priority**: MVP 초기에는 자동 추출 전 수동 입력이
필요하며, 자동 추출 후에도 보정이 필요하다.

**Independent Test**: 관리 화면에서 새 기술 항목을 수동 추가하고,
기존 항목을 수정/삭제한 뒤 메인 목록에 반영되는지 확인.

**Acceptance Scenarios**:

1. **Given** 관리 화면에서, **When** 관리자가 기술 이름, 설명, 카테고리, 출처 영상 링크를 입력하고 저장하면, **Then** 새 항목이 메인 목록에 추가된다
2. **Given** 기존 기술 항목을 선택했을 때, **When** 관리자가 정보를 수정하고 저장하면, **Then** 변경 사항이 메인 목록에 반영된다
3. **Given** 기존 기술 항목을 선택했을 때, **When** 관리자가 삭제를 확인하면, **Then** 해당 항목이 메인 목록에서 제거된다

---

### User Story 5 - 조코딩 채널 자동 동기화 및 분석 (Priority: P1)

관리자는 조코딩 채널에서 AI뉴스 영상을 자동으로 수집하고,
자막 추출 + LLM 분석을 일괄 실행할 수 있다.
Vercel Cron으로 매주 자동 실행되며, 기처리 영상은 제외된다.

**Why this priority**: 수동 URL 입력 없이 채널 전체를 자동으로 관리하여
운영 효율을 극대화하는 핵심 자동화 기능이다.

**Independent Test**: 채널 동기화 버튼 클릭 후 새 영상이 추가되고,
일괄 분석으로 기술 항목이 자동 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 조코딩 채널에 새 AI뉴스 영상이 있을 때, **When** 동기화를 실행하면, **Then** 새 영상 URL이 source_videos에 추가된다
2. **Given** 미분석 영상이 있을 때, **When** 일괄 분석을 실행하면, **Then** 자막 추출 + AI 분석 후 tech_items에 저장된다
3. **Given** Vercel 배포 후, **When** 매주 월요일 09:00 UTC가 되면, **Then** cron이 자동으로 sync+analyze를 실행한다

---

### Edge Cases

- 동일한 AI 기술이 여러 영상에서 중복 소개되었을 때 어떻게 처리하는가?
  - 가정: 중복 항목은 관리자가 수동으로 병합하며, 출처 영상은 복수 등록 가능
- 자막 추출 시 자동 생성 자막의 품질이 낮아 LLM이 기술명을 잘못 추출하면?
  - 가정: 관리자가 자동 추출 결과를 검토/수정 후 확정하는 프로세스
- 조코딩 채널에 AI 뉴스가 아닌 다른 영상이 섞여 있을 때?
  - 가정: 영상 제목에 "AI 뉴스" 키워드 포함 여부로 1차 필터링, 관리자가 최종 선택
- 일괄 분석 중 일부 영상이 실패하면?
  - 가정: 실패한 영상은 건너뛰고 나머지를 계속 처리하며, 실패 건은 별도 표시
- 브라우저 로컬 저장소가 가득 차거나 사용자가 브라우저 데이터를 삭제하면?
  - 가정: 실습 데이터는 유실될 수 있으며, 이에 대한 경고 안내 제공

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 AI 기술 목록을 최신순으로 표시 MUST
- **FR-002**: 각 기술 항목은 이름, 설명, 카테고리, 출처 영상 링크, 소개 날짜를 포함 MUST
- **FR-003**: 사용자는 카테고리별로 기술 목록을 필터링할 수 있어야 한다 MUST
- **FR-004**: 시스템은 YouTube 영상 자막을 추출하고 LLM으로 분석하여 AI 기술의 이름, 설명, 카테고리를 자동 생성 MUST
- **FR-005**: 관리자는 자동 추출된 기술 목록을 검토하고 수정한 후 확정 MUST
- **FR-006**: 관리자는 기술 항목을 수동으로 추가, 수정, 삭제 MUST
- **FR-007**: 사용자는 각 기술 항목에 실습 상태(미시작/진행중/완료)를 설정 MUST. 실습 데이터는 브라우저 로컬 저장소에 보관
- **FR-008**: 사용자는 각 기술 항목에 실습 메모를 작성하고 저장 MUST. 메모는 브라우저 로컬 저장소에 보관
- **FR-009**: 사용자는 실습 상태별로 기술 목록을 필터링할 수 있어야 한다 MUST
- **FR-010**: 각 기술 항목은 하나 이상의 출처 영상 링크를 가질 수 있다 MUST
- **FR-011**: 시스템은 자막이 없는 영상에 대해 명확한 오류 메시지를 표시 MUST
- **FR-012**: 시스템은 환경변수 기반 비밀번호로 관리자 접근을 보호 MUST. 별도 관리자 경로에서 비밀번호를 입력하여 관리 기능에 접근한다. 일반 사용자는 인증 없이 목록 조회 및 실습 추적이 가능하다
- **FR-013**: 사용자는 기술명 또는 설명에 포함된 키워드로 텍스트 검색할 수 있어야 한다 MUST
- **FR-014**: 관리자는 여러 영상 URL을 일괄 입력하여 순차적으로 분석할 수 있어야 한다 MUST
- **FR-015**: 카테고리는 AI 분석 시 동적으로 생성. 기존 카테고리와 매칭하고, 없으면 신규 생성 MUST
- **FR-016**: 시스템은 조코딩 채널에서 AI뉴스 영상을 자동 스크래핑하여 수집 MUST
- **FR-017**: 각 기술 항목은 공식 홈페이지 URL을 포함 SHOULD (AI가 지식 기반으로 조사)
- **FR-018**: 카테고리는 AI 분석 시 동적으로 생성되며, 기존 카테고리와 매칭 MUST
- **FR-019**: 기술 목록은 출처 영상별로 그룹핑하여 최신순 표시 MUST, 카테고리 필터 병행
- **FR-020**: 시스템은 매주 자동으로 새 영상을 수집하고 분석 SHOULD (Vercel Cron)

### Key Entities

- **Tech Item (기술 항목)**: 추출/등록된 개별 AI 기술. 이름, 설명, 카테고리, 소개 날짜, 등록 날짜를 포함. 실습 상태와 메모는 사용자 브라우저 로컬에 별도 저장
- **Source Video (출처 영상)**: 기술이 소개된 YouTube 영상. 영상 URL, 제목, 게시 날짜를 포함. 하나의 기술 항목에 여러 출처 영상이 연결될 수 있음
- **Category (카테고리)**: 기술의 분류. 초기 7개: 이미지 생성, 영상 생성, 음성/음악 AI, 코딩 도구, 챗봇/에이전트, 생산성 도구, 기타. 관리자가 추가 가능
- **Extraction Job (추출 작업)**: 자막 추출 및 LLM 분석 작업의 기록. 대상 영상 URL(단건 또는 복수), 실행 일시, 추출 결과, 상태(진행중/완료/실패)를 포함

### Assumptions

- 조코딩 채널의 AI 뉴스 영상 대부분은 한국어 자막(자동 생성 포함)이 존재한다
- 카테고리는 사전 정의된 7개 목록으로 시작하되, 관리자가 추가할 수 있다
- 한 명의 관리자(프로젝트 소유자)가 사이트를 운영한다
- 일반 사용자는 인증 없이 목록 조회 및 실습 추적이 가능하다 (브라우저 로컬 저장)
- LLM 분석 결과는 100% 정확하지 않으므로 관리자 검토가 필수이다
- LLM은 기술명, 설명, 카테고리를 자동 추출하며, 난이도는 관리자가 수동 설정한다

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 메인 페이지 접속 후 3초 이내에 기술 목록을 확인할 수 있다
- **SC-002**: 관리자는 영상 URL 입력부터 기술 목록 추출 확인까지 2분 이내에 완료할 수 있다
- **SC-003**: 자동 추출된 기술 목록의 정확도가 80% 이상이다 (관리자 수정 비율 20% 미만)
- **SC-004**: 사용자는 기술 항목의 실습 상태 변경을 2번의 클릭 이내로 완료할 수 있다
- **SC-005**: 사이트는 모바일 환경에서도 모든 핵심 기능을 사용할 수 있다
- **SC-006**: 사용자는 검색 키워드 입력 후 1초 이내에 필터링된 결과를 확인할 수 있다
